# Desktop Agents 干活使用指南

> 持续更新文档。最后更新：2026-05-23

这个文档记录如何让 Desktop Agents 不只是聊天，而是帮你实际做事：读文件、看项目、执行命令、截图、多人讨论方案等。

## 1. 启动项目

在项目目录运行：

```powershell
py F:\xmlg\agent\desktop-agents\main.py
```

启动后会出现多个桌面 Agent。第一次使用时，如果没有配置 API Key，会先弹出配置窗口。

## 2. 配置大模型 Key

### 首次配置

第一次启动时会自动检查是否已有 Key：

- 如果存在 `LLM_API_KEY` 或 `DEEPSEEK_API_KEY`，直接启动。
- 如果没有，会弹出“首次使用配置”。

需要填写：

| 字段 | 示例 |
| --- | --- |
| Provider | `deepseek` / `openai` / `claude-compatible` |
| Base URL | `https://api.deepseek.com/v1` |
| Model | `deepseek-chat` |
| API Key | 你的模型 API Key |

### 随时修改

右键任意 Agent，点击：

```text
API Key 设置
```

托盘菜单里也有同样入口。

保存后会热更新当前 Agent 的模型客户端，后续聊天直接使用新配置。

## 3. 基本交互方式

### 和 Agent 说话

左键点击 Agent，输入你想说的话。

例如：

```text
帮我看看这个项目现在有哪些功能
```

Agent 会在桌面气泡里回复。

### 打开群聊记录

右键 Agent，点击：

```text
聊天记录
```

可以查看最近的多 Agent 对话。

### 多 Agent 自动群聊

项目启动后，多个 Agent 会自动进行轻量群聊。你可以随时插话，例如：

```text
大家帮我分析一下下一步应该优先做什么
```

系统会选择 1-2 个 Agent 回复。

## 4. 让 Agent 真正干活

现在 Agent 支持工具调用。你只需要自然语言说明任务，模型会通过 Function Calling 判断是否需要工具。

### 4.1 读项目文件

示例：

```text
帮我读取 F:\xmlg\agent\desktop-agents\config.py，看一下模型配置现在怎么加载
```

会触发工具：

```text
read_file
```

权限级别：`NORMAL`

你会看到确认弹窗，允许后 Agent 才能读取文件。

注意：

- 默认只允许读项目目录内的文件。
- `.env`、密钥文件、证书文件等敏感文件会被拒绝读取。
- 文件内容会限制大小，过大的内容会截断。

### 4.2 执行命令

示例：

```text
帮我运行 py --version 看看 Python 版本
```

会触发工具：

```text
execute_command
```

权限级别：`DANGEROUS`

每次执行命令前都会弹出高危确认。

当前命令执行限制：

- 不接受 shell 字符串。
- 使用结构化参数：`program + args`。
- 不使用 `shell=True`。
- 工作目录必须在允许目录内。
- 有超时时间。
- stdout/stderr 会限制大小。

适合让 Agent 做：

```text
帮我运行测试
帮我检查 Python 版本
帮我编译项目看看有没有语法错误
帮我列出当前项目依赖
```

不建议让 Agent 做：

```text
删除文件
清空目录
修改系统配置
安装未知依赖
运行网上复制的命令
```

### 4.3 截图

示例：

```text
帮我截个图，看看现在桌面上应用状态
```

会触发工具：

```text
screenshot
```

权限级别：`NORMAL`

当前截图能力会保存截图文件并返回路径和尺寸信息。注意：当前模型主要是文本模型，不能直接理解图片内容；后续可以接入视觉模型或 OCR。

## 5. 权限等级说明

| 权限 | 是否需要确认 | 示例 |
| --- | --- | --- |
| SAFE | 不需要 | 纯内部计算、格式化等 |
| NORMAL | 需要确认 | 读文件、截图 |
| DANGEROUS | 高危确认 | 执行命令 |

原则：

- Agent 想使用工具时，先由模型提出工具调用。
- 工具执行器检查权限。
- 需要确认的操作会弹窗。
- 你拒绝后，工具不会执行，Agent 会收到“用户拒绝”的结果。

## 6. 推荐用法

### 6.1 让 Agent 看代码

```text
帮我读一下 core/agent.py，总结 Agent 是怎么生成回复的
```

### 6.2 让 Agent 检查某个功能

```text
帮我检查 API Key 设置流程相关代码，看看启动时是不是会先检测 Key
```

### 6.3 让 Agent 跑测试

```text
帮我运行这个项目的单元测试
```

它可能会请求执行命令，你确认后再运行。

### 6.4 让多个 Agent 讨论方案

```text
大家讨论一下，如果我要让 Agent 支持长期任务队列，应该怎么设计？
```

适合用来做：

- 产品方案讨论
- 技术方案对比
- 风险评估
- 下一步优先级判断

### 6.5 让 Agent 分角色工作

你可以给不同 Agent 不同人格，例如：

- 产品经理：拆需求、看体验
- 架构师：看整体设计
- 测试工程师：找边界条件
- 安全工程师：看权限和风险

然后让他们讨论：

```text
产品、架构、测试分别说一下这个功能还缺什么
```

## 7. 导入聊天记录人格

右键 Agent，点击：

```text
导入聊天记录人格
```

支持：

- 单文件导入：`导入聊天记录文件`
- 文件夹导入：`批量导入`

支持格式：

```text
txt / csv / json / sqlite / db / sqlite3
```

如果不知道怎么导出聊天记录，界面里会提示参考：

```text
https://github.com/r266-tech/wechat-local-mcp
```

项目本身不自动下载第三方工具。

## 8. 常见任务示例

### 让 Agent 总结项目

```text
帮我读一下项目核心文件，总结当前 Desktop Agents 已经支持哪些能力
```

### 让 Agent 做一次健康检查

```text
帮我检查项目能不能编译，能不能跑测试
```

### 让 Agent 查配置

```text
帮我看一下现在模型 Provider、Base URL、Model 是怎么保存和加载的
```

### 让 Agent 看 UI 问题

```text
帮我检查导入聊天记录弹窗的按钮文案是否清楚
```

### 让 Agent 给重构建议

```text
大家讨论一下，现在工具执行能力还可以怎么优化，但先不要改代码
```

## 9. 安全建议

- 不要让 Agent 执行你看不懂的命令。
- 遇到 `DANGEROUS` 权限弹窗时，先确认命令内容。
- 不要把 API Key、私钥、密码文件作为任务内容发给 Agent。
- 如果 Agent 请求读取敏感文件，直接拒绝。
- 当前命令执行不是系统级强沙箱，只是应用层限制；高风险命令仍需谨慎。
