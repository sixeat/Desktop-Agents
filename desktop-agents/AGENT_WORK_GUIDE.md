# Desktop Agents 使用指南

> 持续更新文档。最后更新：2026-05-24

Desktop Agents 是一个轻量桌面多 Agent 聊天应用。项目核心目标是从你和朋友的聊天记录中蒸馏出“我”或“朋友”的人格 Agent，并用桌面宠物/头像形象承载这些人格；宠物不是产品本体，而是人格 Agent 的默认可视化形象，后续可拓展为自定义宠物形象。

## 1. 产品目标

当前主线不是做纯宠物应用，而是：

1. 从聊天记录里分析一个人的说话习惯、常聊话题、口头禅和语气。
2. 把这些特征蒸馏成人格 Agent。
3. 让人格 Agent 以桌面宠物/头像形象常驻桌面。
4. 支持你和单个 Agent 聊天，也支持多个 Agent 自动群聊。
5. 后续拓展自定义宠物形象，让不同人格可以选择不同外观。

## 2. 启动项目

默认启动宠物形象承载的 Agent 模式：

```powershell
py F:\xmlg\agent\desktop-agents\main.py
```

启动后会读取已保存的 Agent 配置；首次没有 Agent 时会弹出新建 Agent 窗口：

1. 为单个 Agent 设置名字。
2. 选择基础类型和性格标签。
3. 可上传 6 张情绪 PNG：正常、开心、难过、困、惊讶、生气。
4. 只上传“正常”PNG 也可以，其他情绪会复用正常图。
5. 确认后桌面出现这个 Agent，后续可在托盘 `Agent 管理` 中继续新建、编辑、删除或导入人格。

PNG 是静态图片，但桌面端会叠加呼吸、弹跳、摇摆、下沉、抖动等情绪动作，让形象看起来是动态的。

默认模式不会强制配置 API Key。没有 Key 时使用本地回复；配置 Key 后，用户聊天和 Agent 自动群聊都会使用配置好的 LLM。

旧版头像 Agent 模式仍保留：

```powershell
py F:\xmlg\agent\desktop-agents\main.py --mode agents
```

## 3. API Key 与模型配置

### 默认模式

默认模式不会因为没有 API Key 阻塞启动。需要配置时，右键系统托盘图标，点击：

```text
API Key 设置
```

保存后，后续单 Agent 聊天和自动群聊会使用新配置。

### 旧版 `--mode agents`

旧版头像 Agent 模式启动时会检查 API Key：

- 如果存在 `LLM_API_KEY` 或 `DEEPSEEK_API_KEY`，直接启动。
- 如果没有，会弹出“首次使用配置”。

可配置字段：

| 字段 | 示例 |
| --- | --- |
| Provider | `deepseek` / `openai` / `claude-compatible` |
| Base URL | `https://api.deepseek.com/v1` |
| Model | `deepseek-chat` |
| API Key | 你的模型 API Key |

配置持久化策略：

- Provider / Base URL / Model 保存到 `QSettings`。
- API Key 优先保存到系统凭据管理器。
- 如果系统凭据管理器不可用，会 fallback 到当前 Windows 用户配置。
- 不写入项目 `.env`。

## 4. 桌面 Agent 基本交互

### 拖动和点击

- 按住左键拖动桌面 Agent，可以移动到桌面任意位置。
- 左键点击桌面 Agent，会触发 happy 情绪和一次本地互动回复。
- 桌面 Agent 有轻微呼吸动画，情绪变化时会弹跳。
- 自定义 PNG 形象会按当前情绪切换图片，并叠加开心摇摆、难过下沉、困倦轻晃、惊讶上跳、生气抖动等动作。
- 导入 PNG/JPG 形象时会自动把和图片边缘连通的近白色背景抠成透明，保留角色内部白色细节。

### 和单个 Agent 说话

双击某个桌面 Agent，会打开独立聊天窗口，可以像普通对话窗口一样输入消息并发送。

右键某个桌面 Agent，点击：

```text
和它说话
```

也可以通过旧的输入弹窗发送单条消息。

发送消息后：

- 该 Agent 会根据输入更新情绪。
- 桌面气泡会显示回复。
- 聊天窗口会同步显示你和该 Agent 的消息。
- 没有 API Key 时走本地回复。
- 有 API Key 时走配置好的 LLM。
- 这段对话会进入该 Agent 自己的单聊记录。

### 打招呼和切换情绪

右键某个桌面 Agent 可以：

- `打个招呼`：让它本地回应一句。
- `切换情绪`：手动切换 normal / happy / sad / sleepy / angry / surprised。
- `退出`：退出应用。

## 5. 群聊记录和单 Agent 记录

当前有两类聊天记录，互相独立。

### 群聊记录：你和 Agent 们

右键系统托盘图标，点击：

```text
群聊记录
```

可以查看：

- Agent 之间自动群聊的消息。
- 群聊通道中的 Agent 回复。
- 最近 50 条群聊记录。

自动群聊记录只写入群聊窗口，不会写入某个 Agent 的单聊记录。

### 单 Agent 记录：你和某个 Agent

右键某个桌面 Agent，点击：

```text
聊天记录
```

可以查看：

- 你发给这个 Agent 的消息。
- 这个 Agent 对你的回复。
- 点击/打招呼产生的单 Agent 互动。
- 最近 50 条单聊记录。

每个 Agent 的单聊记录分开保存，互不串扰。

## 6. 多 Agent 自动群聊

默认模式启动后，如果桌面上有至少 2 个 Agent，会自动进行轻量群聊：

- 首次自动群聊会在启动后短暂延迟触发。
- 后续会按固定间隔轮流让 Agent 发言。
- 没有 API Key 时使用本地群聊话术。
- 有 API Key 时使用配置好的 LLM 生成聊天内容。
- 群聊中的情绪会影响其他 Agent 的状态。

## 7. 人格与情绪

当前桌面 Agent 支持：

- 默认人格：没有人格文件时，按宠物类型生成默认人格。
- 人格标签：活泼、温柔、毒舌、沉稳。
- 人格 Prompt：包含名字、类型、性格、口头禅、话题和语气约束。
- 情绪识别：根据用户输入识别 happy / sad / sleepy / angry / surprised。
- 人格 × 情绪联动：同一句输入在不同人格和情绪下会有不同回复风格。

## 8. 聊天记录人格导入

### 默认桌面 Agent 模式

右键某个桌面 Agent，点击：

```text
导入人格
```

流程：

1. 选择微信聊天记录导出文件。
2. 支持 `.json` / `.txt` / `.csv`。
3. 输入 `聊天记录里的联系人`，用于定位导出文件中要分析的发言者。
4. 输入 `导入后 Agent 名称`，用于导入后的桌面显示名和聊天窗口标题。
5. 点击 `分析`，预览性格、口头禅、常聊话题、句式、emoji 和平均句长。
6. 点击 `确认导入` 后生成 `models/pet_personas/<Agent 名字>/persona.json`。
7. 当前桌面 Agent 会立即切换到新人格并说一句问候；如果聊天窗口已打开，窗口标题会同步更新。

`persona.json` 只保存蒸馏后的人格分析结果和 prompt，不保存原始聊天记录。

### 旧版头像 Agent 模式

旧版头像 Agent 模式中仍保留原有聊天记录人格导入能力。

右键 Agent，点击：

```text
导入聊天记录人格
```

支持：

- 单文件导入：`导入聊天记录文件`
- 文件夹导入：`批量导入`

支持格式：

```text
txt / csv / json / sqlite / db / sqlite3
```

项目本身不自动下载第三方工具；导入界面只提供参考链接。

## 9. 系统托盘

默认模式的系统托盘包含：

- API Key 设置
- Agent 管理
- 群聊记录
- 批量导入人格
- 显示萌宠
- 隐藏萌宠
- 退出

全局能力放在托盘里；单个 Agent 的聊天、单聊记录和情绪切换保留在对应 Agent 的右键菜单里。

## 10. 当前已移除能力

当前精简版已移除以下过度设计能力：

- 工具执行框架
- 权限分级系统
- 工具权限确认弹窗
- 读文件工具
- 执行命令工具
- 截图工具
- 模型 tool_calls 调用循环

因此，桌面 Agent 现在只负责聊天、人格和情绪交互，不会主动读取文件、执行命令或截图。

## 12. 安全建议

- 不要把 API Key、私钥、密码文件内容直接发给 Agent。
- 当前版本不会主动读取本地文件，也不会执行本地命令；人格导入只读取你在文件选择器里明确选择的聊天记录导出文件。
- API Key 优先保存到系统凭据管理器；如果不可用，会 fallback 到当前 Windows 用户配置。

## 13. 后续可继续优化

可以继续考虑：

- 聊天记录搜索、导出或持久化。
- 自动群聊暂停/频率设置。
- 人格导入的大文件分析改为后台线程，避免 UI 短暂卡顿。
- 是否保留旧版头像 Agent 的聊天记录导入人格。
- 是否减少默认 Agent 数量。
- 是否整理人格配置文件。
