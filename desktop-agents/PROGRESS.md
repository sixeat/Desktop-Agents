# Desktop Agents 进度记录

> 更新时间：2026-05-24

## 当前定位

Desktop Agents 当前定位为轻量桌面多 Agent 聊天应用。项目核心目标是从用户和朋友的聊天记录中蒸馏出“我”或“朋友”的人格 Agent，再用桌面宠物/头像形象作为人格承载和交互入口。宠物表现层不是产品本体，而是人格 Agent 的默认桌面形象；后续可拓展自定义宠物形象。

`v0.1` 已作为备选版本保留；后续精简完成并确认 1.0 范围后再提交、打标签和推送。

## 今日完成

### 1. 桌面人格 Agent 默认模式

- 默认启动进入桌面人格 Agent 模式。
- 当前版本用内置宠物形象作为人格 Agent 的默认载体。
- 默认启动会读取已保存的 Agent 配置。
- 首次启动没有 Agent 时弹出新建 Agent 窗口。
- 支持单个 Agent 单独新建、编辑、删除和管理。
- 新建/编辑时可设置名字、基础类型、性格标签和 6 张情绪 PNG 形象。
- 6 张情绪形象对应：正常、开心、难过、困、惊讶、生气。
- 只配置正常 PNG 时，其他情绪会自动复用正常图。
- 头像图片会自动把和图片边缘连通的近白色背景抠成透明。
- 桌面显示用户创建的 Agent，不再强制固定 3 个不同类型。
- 桌面 Agent 支持拖动、点击、呼吸动画、情绪切换、情绪动作和桌面气泡。
- 旧版头像 Agent 模式保留在 `--mode agents`。

### 2. 人格训练、默认人格和情绪系统

- 新增人格训练器，可从聊天文本分析口头禅、句式、emoji、话题、平均句长和打招呼风格。
- 当前基础能力已能把聊天文本蒸馏为人格档案；后续需要继续完善“我/朋友”人格导入和选择流程。
- 无人格文件时按形象类型生成默认人格。
- System Prompt 包含名字、类型、性格、口头禅、话题和语气约束。
- 新增情绪引擎，支持 happy / sad / sleepy / angry / surprised。
- 用户输入会触发情绪变化并影响宠物表情。
- 人格和情绪会共同影响本地回复风格。

### 3. LLM 聊天接入

- 默认桌面 Agent 模式不强制配置 API Key。
- 没有 API Key 时使用本地回复。
- 配置 API Key 后，单 Agent 聊天和自动群聊都使用配置好的 LLM。
- API Key 设置入口放在系统托盘右键菜单。
- LLM 调用在后台线程执行，完成后回到 UI 线程更新气泡、情绪和聊天记录。
- LLM 异常时回退到本地回复，不展示技术错误。

### 4. 群聊/单 Agent 详细记录

- 默认桌面 Agent 模式新增群聊记录窗口。
- 每个桌面 Agent 新增独立单聊记录窗口。
- 群聊和单聊分开保存，互不串扰。
- UI 详细记录只保留最新 50 条。
- UI 详细记录与 `PetCompanion.history` 的 LLM 上下文历史分离。
- 系统托盘新增 `群聊记录` 入口。
- 桌面 Agent 右键菜单新增 `聊天记录` 入口。
- LLM 异步回复按 `group` / `direct` 通道写入正确记录。

### 5. 单 Agent 人格导入和聊天窗口

- 默认桌面 Agent 右键菜单新增 `导入人格`。
- 支持选择 `.json` / `.txt` / `.csv` 微信聊天记录导出文件。
- 导入时 `聊天记录里的联系人` 和 `导入后 Agent 名称` 分开填写。
- 导入前可预览性格、口头禅、话题、句式、emoji、平均句长和分析消息数。
- 确认导入后保存 `models/pet_personas/<Agent 名字>/persona.json`。
- 保存内容是蒸馏后的人格档案和 prompt，不保存原始聊天记录。
- 导入后只切换当前选中的桌面 Agent，并立即按新人格说话。
- 双击桌面 Agent 可打开独立单聊窗口，支持输入消息和查看最近单聊记录。
- Agent 改名或导入人格后，已打开的聊天窗口标题会同步刷新。
- 聊天窗口发送消息复用 direct 聊天链路，不污染群聊记录。

### 6. 旧版能力保留和精简

- 旧版 `--mode agents` 模式仍可启动。
- 旧版 `AgentBus` 群聊历史上限调整为 50 条。

- 工具执行、权限分级、截图、tool_calls 循环等过度设计能力保持移除。

## 当前已知限制

1. 聊天记录目前是运行时内存记录，应用退出后清空。
2. 聊天记录暂不支持搜索、导出或置顶。
3. 自动群聊暂不支持暂停/频率设置 UI。
4. 人格导入分析目前同步执行，超大导出文件可能让导入弹窗短暂卡顿。
5. 多模型配置是 OpenAI-compatible 基础配置，没有完整 Provider 预设 UI。
6. API Key fallback 到 QSettings 是为了解决当前 Windows 环境 keyring 后端不可用问题，安全性弱于系统凭据管理器。

## 备注

- 当前版本不会主动读取文件、执行命令或截图；人格导入只读取用户在文件选择器中明确选择的导出文件。
- API Key 不写入项目文件。
- 项目不会主动下载 wx-mcp，只提供参考链接。
- 未经用户确认，不推送到远程。
